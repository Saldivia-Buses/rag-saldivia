### Hallazgos clave

- **Harvey.ai (2025):** Sistemas de ingesta a escala usan tier por tamaño de archivo. Tiny/Small procesan síncronos; Medium/Large van a una cola async con workers. El tier determina timeout, paralelismo y estrategia de retry.
- **Robust Upload System (Medium):** Progress tracking en tiempo real usando SSE o WebSockets. Estado persistido en DB para resumabilidad. Checksum para detectar uploads corruptos.
- **Best Practices (LinkedIn):** Pre-signed URLs para grandes archivos, chunk-based upload con reassembly, idempotent job IDs para dedup en resume.
