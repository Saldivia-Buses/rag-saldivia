## v2.0.2 — Astro Service + Repo Cleanup (2026-04-07)

27 commits, PR #125. New vertical module: astrological calculation engine.

### New Service: `services/astro/`

Full port of Python `astro-v2` to Go with Swiss Ephemeris via CGO. Plan 11 implemented across 17 PRs (#104-#124), then hardened with security audit + gateway review fixes.

**Calculation Engine (11 techniques):**
- Natal chart with golden file verification against Python reference
- Secondary progressions with ingress detection
- Solar arc directions with antiscia
- Primary directions (Polich-Page semi-arc)
- Slow planet transits + station detection (D/Rx/Rx/D)
- Solar/lunar returns via SolcrossUT + Newton iteration
- Profections + firdaria (pure date arithmetic)
- Eclipses + zodiacal releasing + fixed stars
- Context builder + intelligence brief

**HTTP API (17 endpoints):**
- 10 calculation endpoints (natal, transits, solar-arc, directions, progressions, returns, profections, firdaria, fixed-stars, brief)
- Full CRUD contacts (list, search, create, update, delete) with pagination
- SSE query endpoint with true token streaming via LLM

**Infrastructure:**
- Docker Compose wiring with ephemeris data volume
- Traefik routing (`/v1/astro/*` -> port 8011)
- Tool manifest for agent integration (`modules/astro/`)
- Handler integration tests (health, auth, validation)
- sqlc-generated repository with tenant+user scoped queries

### Security Hardening

22 findings from security audit + gateway review resolved:
- **FailOpen split**: `false` for write endpoints, `true` for reads
- **RBAC**: `astro.read` / `astro.write` permissions on all endpoints
- **Prompt injection**: query sanitization (2000 char limit, control char stripping)
- **Proper error handling**: 409 only for unique violations, 500 logged with request context
- **Separate request struct**: CreateContact no longer accepts tenant_id/user_id in body
- **Generic errors**: no contact name reflection in error messages

### Platform Changes

- **`pkg/llm.StreamChat`**: new streaming method on `ChatClient` interface with OpenAI SSE protocol support
- **`moonCrossUT`**: returns error on non-convergence instead of silent skip
- **Solar arc**: mid-year standardized to July 1 (was June 15)

### Repo Cleanup

- Removed `_archive/` (5.5 GB of dead 1.0.x code, 338 tracked files)
- Removed `.cursor/` IDE configs (17 files)
- Both added to `.gitignore`

## 2026-04-03 -- Hardening & Production Readiness

Completed all 16 phases of the hardening plan (`docs/plans/2.0.x-plan01-sda-framework.md`).

### Security

- JWT migrated from HS256 to Ed25519 asymmetric signing (all 8 services)
- RBAC permission enforcement on all endpoints (chat: `chat.read`/`chat.write`, rag: `collections.read`, ingest: `ingest.write`)
- MFA (TOTP) support: setup, verify, disable flows in auth service
- Audit logging with action format validation (`service.entity.verb`) and composite index
- Security headers (`SecureHeaders` middleware) on all API responses
- Slug cross-validation: tenant slug from JWT must match request context, prevents token replay between tenants
- Platform admin check: distinguishes platform admins from tenant admins via slug claim
- Brute force protection: progressive lockout on failed logins, permanent lockout after 20 attempts
- DB credentials: encrypted columns added to platform DB (`postgres_url_enc`, `redis_url_enc`) for AES-256-GCM at rest
- Request body limits: `MaxBytesReader` on all POST/PUT handlers (1MB default, 100MB for ingest uploads)

### Infrastructure

- Production Docker Compose with secrets, resource limits, HTTPS
- NATS per-service authorization
- Docker network segmentation (frontend/backend/data tiers)
- CrowdSec IDS integration for Traefik
- CI security scanning pipeline (gosec + trivy)
- OpenTelemetry instrumentation on all 8 services (`otelhttp` handler wrapping)

### Backend

- All services migrated to sqlc + repository layer
- 8 Go microservices fully operational: auth, ws, chat, rag, notification, platform, ingest, feedback
- Feedback service: central nervous system with NATS consumer, hourly aggregator, health scores, alerter
- Chat service: extended thinking support (`thinking` column), source citations, metadata
- Ingest service: async pipeline with JetStream worker, Blueprint forwarding, tenant-namespaced collections
- Notification service: JetStream durable consumer, user preference-aware delivery, email + in-app + WS push
- Platform service: full CRUD for tenants/modules/flags/config, platform admin auth middleware

### Frontend

- Chat enhancements: model selector, reasoning display, markdown rendering, URL routing
- Auth: redirect fixes, CORS configuration, streaming persistence
- Feedback integration: quality dashboard endpoints available

### Documentation

- Comprehensive README for all 8 services
- Platform CHANGELOG
- ADR-013: Go microservices migration
- ADR-014: Ed25519 JWT signing
- ADR-015: Database-per-tenant isolation
