SDA Framework Architecture

Multi-tenant SaaS platform — Hybrid Cloud/InHouse deployment — 15 Go microservices + GPU pipeline

Go Microservices (15)

  • • chi router + sqlc + slog
  • • JWT Ed25519 local validation
  • • Per-tenant DB/Redis isolation
  • • NATS async events on every write
  • • OpenTelemetry traces + metrics
  • • gRPC for internal service calls
  • • Feature flags per tenant

GPU / ML Pipeline

  • • RTX PRO 6000 (96GB VRAM)
  • • SGLang: OCR (PaddleOCR-VL 1.5)
  • • SGLang: Vision (Qwen3.5-9B)
  • • Tree RAG (PageIndex-inspired, no vectors)
  • • NVIDIA API: Nemotron-Super-49B
  • • Model-agnostic via OpenAI-compatible API
  • • Slot-per-pipeline-step architecture

Data & Infrastructure

  • • PostgreSQL 16: platform + per-tenant DBs
  • • Redis 7: cache, sessions, rate limits
  • • NATS + JetStream: persistent events
  • • MinIO: S3-compatible object storage
  • • Traefik v3: TLS, subdomain routing
  • • CrowdSec IDS + Docker autoheal
  • • 4 Docker networks (frontend/backend/data/lan)

Security

  • • JWT + refresh tokens (Ed25519)
  • • RBAC with granular permissions
  • • Tenant isolation at every layer
  • • AES-256-GCM envelope encryption
  • • Cloudflare Tunnel (no exposed ports)
  • • CrowdSec IDS on Traefik logs
  • • Per-service NATS auth tokens

Frontend

  • • Next.js 16.2.2 + React 19.2.4 + TypeScript
  • • Tailwind CSS 4 + shadcn/ui (web only)
  • • TanStack Query v5 for state
  • • WebSocket-first real-time
  • • Module Registry (dynamic imports)
  • • Login app: minimal, no shadcn
  • • Deploy: Vercel or Fly.io

Observability

  • • OpenTelemetry Collector (gRPC + HTTP)
  • • Grafana 11.6 (single pane of glass)
  • • Tempo: distributed tracing
  • • Prometheus v3.4: metrics (30d retention)
  • • Loki + Promtail: log aggregation
  • • AlertManager → Notification webhook
  • • HealthWatch + AI triage
