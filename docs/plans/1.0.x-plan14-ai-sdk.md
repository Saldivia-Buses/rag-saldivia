## Contexto

El chat usa SSE manual: `useRagStream` (147 líneas, complejidad 19) hace fetch,
lee el stream con `getReader()`, parsea líneas `data: {...}` manualmente, y
extrae `delta.sources` (citations) del mismo stream. Es frágil y no estándar.

El Vercel AI SDK (`ai`) es el estándar de la industria para streaming de IA en
Next.js. Provee `useChat` (client), `createUIMessageStream` (server), y
`@ai-sdk/openai-compatible` para backends OpenAI-compatible como el RAG Blueprint.

Enzo decidió adoptar AI SDK (2026-03-29). Es base para json-render (Plan 17).

## Enfoque

**No usar `@ai-sdk/openai-compatible` como provider directo** porque la route
tiene mucha lógica pre-stream (auth, rate limit, RBAC, focus modes, memory,
project context, language detection) que necesita correr antes del fetch al RAG.

**Usar `createUIMessageStream`** para transformar el stream NVIDIA al protocolo
AI SDK. El server sigue haciendo el fetch manual al RAG pero empaqueta la
respuesta en formato AI SDK. El client usa `useChat` que entiende el protocolo
nativamente.

**Citations via `source` parts** — el AI SDK tiene soporte nativo para RAG
sources. Las pasamos como `source-document` parts en el stream.

`ai-stream.ts` es el puente entre el SSE de NVIDIA y el protocolo AI SDK:

```typescript
// Pseudocódigo del adapter
import { createUIMessageStream, createUIMessageStreamResponse } from "ai"
import { parseSseLine } from "./stream"
import { CitationSchema } from "@rag-saldivia/shared"

export function createRagStreamResponse(
  ragStream: ReadableStream,
  contentType: string
) {
  const stream = createUIMessageStream({
    execute: async ({ writer }) => {
      const reader = ragStream.getReader()
      const decoder = new TextDecoder()

      // Leer SSE de NVIDIA, escribir en AI SDK protocol
      while (true) {
        const { done, value } = await reader.read()
        if (done) break
        const chunk = decoder.decode(value, { stream: true })

        for (const line of chunk.split("\n")) {
          // Extraer token de texto
          const token = parseSseLine(line)
          if (token) {
            writer.write({ type: "text-delta", textDelta: token })
          }

          // Extraer citations
          if (line.startsWith("data: ")) {
            // parsear delta.sources → writer.write({ type: "source", ... })
          }
        }
      }
    },
  })

  return createUIMessageStreamResponse({ stream })
}
```

## Riesgos

| Riesgo | Mitigación |
|---|---|
| `useChat` no soporta nuestro formato de body | `body` param acepta cualquier objeto, se envía como JSON |
| Citations en `delta.sources` no se parsean bien | El adapter las extrae antes de pasar al protocol |
| Mock mode no funciona con AI SDK | El adapter lee SSE genérico, mock produce SSE válido |
| `onFinish` de `useChat` no tiene las sources | Pasamos sources como `source` parts en el stream |
| ChatInterface pierde funcionalidad | Solo pierde lo archivado (artifacts, voice, etc.) |
