## Qué se hizo

Migración del chat de SSE manual (`useRagStream`, 147 líneas, complejidad 19) al Vercel AI SDK estándar de la industria. Se usa `createUIMessageStream` en el server y `useChat` en el client.

### Enfoque

**No se usó `@ai-sdk/openai-compatible` como provider directo** porque la route tiene mucha lógica pre-stream (auth, rate limit, RBAC, focus modes, memory). En su lugar, el server sigue haciendo el fetch manual al RAG pero empaqueta la respuesta en formato AI SDK via `createUIMessageStream`.

**Citations via `source` parts** — las fuentes del RAG se pasan como `source-document` parts nativos del AI SDK.

### Archivos modificados

- `apps/web/src/app/api/rag/generate/route.ts` — transformar respuesta a AI SDK stream
- `apps/web/src/components/chat/ChatInterface.tsx` — `useChat` reemplaza hook manual
- `apps/web/src/components/chat/SourcesPanel.tsx` — adaptado a source parts
- **Nuevo:** `apps/web/src/lib/rag/ai-stream.ts` — adapter NVIDIA SSE → AI SDK Data Stream
- **Archivado:** `useRagStream.ts`, `detect-artifact.ts` → `_archive/`

### Resultado

- `useRagStream` (147 líneas) → eliminado, reemplazado por `useChat` (estándar)
- `ai-stream.ts` — adapter limpio de ~117 líneas
- Citations nativas del AI SDK
