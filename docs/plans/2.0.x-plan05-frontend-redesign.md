# Plan 05 — Frontend Redesign (SDA Identity)

> **Prerequisito:** Plan 02 (Full Wiring) completado — frontend funcional al 100%
> **Sprint:** think > plan > execute > review > test > ship
> **Servicios afectados:** web (frontend only)

## Contexto

El frontend funciona completamente (chat, dashboard, notifications, settings, auth,
WebSocket, modules) pero tiene aspecto de template generico de shadcn con 42 temas
tweakcn, fonts excesivas, y cero identidad propia. Se necesita un rediseno premium
inspirado en Linear y Adobe Spectrum: dark-first, identidad SDA, dos modes
curados (SDA Dark + SDA Light).

**Principios de diseno:**
- "Don't compete for attention you haven't earned" -- sidebar dimmer que content
- "Structure should be felt, not seen" -- menos borders, mas spacing
- Dark-first: el modo oscuro es el default y la experiencia principal

### Fase 1: Theme Foundation

**Objetivo:** Reemplazar el sistema de 42 temas con tokens SDA fijos. Despues de esta
fase, la app se ve con la paleta SDA en dark y light, aunque los componentes aun no
esten restyled.

**Archivos eliminados:**

**Archivos reescritos:**
- `apps/web/src/app/globals.css`

**Archivos modificados:**
- `apps/web/src/app/layout.tsx`
- `apps/web/src/components/dark-mode-toggle.tsx`
- `apps/web/src/app/(core)/system-settings/page.tsx`

**Cambios detallados:**

1. **Eliminar `apps/web/src/lib/theme-presets.ts`** -- borrar el archivo completo (42 temas tweakcn).

2. **Eliminar `apps/web/src/components/theme-selector.tsx`** -- borrar el archivo completo (grid de seleccion de temas).

3. **Reescribir `apps/web/src/app/globals.css`** -- reemplazar las variables oklch actuales con los tokens SDA hex convertidos a oklch. Mantener la estructura `@import`, `@custom-variant`, `:root` (SDA Light), `.dark` (SDA Dark), `@theme inline`, `@layer base`. Cambios:
   - `:root` = paleta SDA Light completa (todos los tokens: background, foreground, card, popover, primary, secondary, muted, accent, destructive, border, input, ring, chart-1..5, sidebar-*)
   - `.dark` = paleta SDA Dark completa
   - `--font-sans`: `var(--font-inter), sans-serif`
   - `--font-mono`: `var(--font-jetbrains-mono), monospace`
   - `--font-serif`: `var(--font-lora), serif`
   - `--radius`: `0.5rem` (Linear-like, no rounded-xl)
   - Shadows: simplificar a sutiles (near-zero spread, low opacity)
   - Agregar clase `.dark` por default via `@layer base` a html

4. **Reescribir `apps/web/src/lib/fonts.ts`** -- reducir de 25 fonts a 3:
   - `Inter` (sans -- UI principal)
   - `JetBrains_Mono` (mono -- code blocks)
   - `Lora` (serif -- decorativo si se usa)
   - Mantener exports: `fontVariables`, `fontClassNames`
   - Eliminar `fontFamilyMap` (ya no se necesita sin theme-presets)

5. **Reescribir `apps/web/src/lib/theme-provider.tsx`** -- simplificar drasticamente:
   - Eliminar imports de `theme-presets` y `fontFamilyMap`
   - Eliminar funciones `applyTheme`, `resolveFontVar`, `allThemeVars`, `defaults`
   - Nuevo context API (cambia la interface publica):
     ```typescript
     // ANTES: { themeId, theme, setTheme, mode, setMode, ... }
     // DESPUES:
     interface ThemeContext {
       mode: "dark" | "light";
       setMode: (m: "dark" | "light") => void;
     }
     export function useTheme(): ThemeContext
     ```
   - `setMode` toggle `document.documentElement.classList` y persiste en `localStorage("sda-dark-mode")`
   - Default mode: `"dark"` (dark-first)
   - Eliminar el evento custom `sda-mode-change`
   - Eliminar `sda-theme` y `sda-theme-vars` de localStorage (ya no existen)

6. **Modificar `apps/web/src/app/layout.tsx`**:
   - Simplificar `themeScript`: solo chequear `sda-dark-mode` en localStorage.
     Logica: `if (localStorage.getItem("sda-dark-mode") !== "false") classList.add("dark")`.
     Resultado: first-time visitors (key no existe) → dark. Solo light si el
     usuario explicitamente eligio light. Esto es dark-first.
   - Eliminar la clase `.dark` del `@layer base` de globals.css (el script la maneja).
   - Reducir `fontVariables` de 25 clases a 3 (viene automatico del cambio en fonts.ts)
   - Eliminar el `div` preloader de font classNames (con 3 fonts no hace falta)
   - Mantener `ThemeProvider` wrapper pero ahora es el simplificado

7. **Reescribir `apps/web/src/components/dark-mode-toggle.tsx`**:
   - Cambiar de `Button` icon (Sun/Moon) a `Switch` component de shadcn
   - Importar `useTheme` del nuevo theme-provider
   - El Switch muestra on = dark, off = light
   - Exportar como `DarkModeSwitch` (renombrar)
   - Mantener export alias `DarkModeToggle` para no romper imports existentes temporalmente

8. **Modificar `apps/web/src/app/(core)/system-settings/page.tsx`**:
   - Eliminar `import { ThemeSelector }` y `import { DarkModeToggle }`
   - Importar `useTheme` del nuevo theme-provider y `Switch` de shadcn
   - Eliminar la seccion `<Separator>` + `<ThemeSelector />` al final
   - Reemplazar `<DarkModeToggle />` con un `<Switch>` inline que usa `useTheme().setMode()`
   - Layout: quitar la seccion de Separator + ThemeSelector

**Verificacion:**
- [ ] `cd apps/web && bun run build` compila sin errores
- [ ] No quedan imports a `theme-presets`, `theme-selector`, `fontFamilyMap`
- [ ] `localStorage.getItem("sda-dark-mode")` controla dark/light
- [ ] Default es dark mode
- [ ] Switch en system-settings togglea correctamente
- [ ] Colores SDA visibles en toda la app (background near-black en dark)
- [ ] Solo 3 fonts se cargan (verificar Network tab)

**Commit:** `feat(web): replace tweakcn theme system with SDA identity tokens`
