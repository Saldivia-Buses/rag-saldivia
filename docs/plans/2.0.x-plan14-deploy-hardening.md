# Plan 14 — Deploy Hardening: All-Terrain + Self-Healing + Full Observability

> **Prerequisito:** Plan 13 mergeado (astro v2 sync)
> **Fecha:** 2026-04-09
> **Estado:** pendiente
> **Deadline:** deploy semana del 14 de abril
> **Servicios afectados:** `deploy/`, `pkg/`, `services/*/cmd/`, `Makefile`
> **Scope:** Tres objetivos — (A) configuracion portable, (B) deploy robusto que
>   se repara solo, (C) observabilidad data-rich donde nada se escapa
> **Fuera de scope:** BigBrother (network intelligence) → Plan 15 separado

---

## Problema

**A) Configuracion acoplada al hardware:**
`sda.app` hardcodeado en 15+ lugares, GPU index fijo, paths absolutos, credenciales
dev en produccion. Cambiar de maquina requiere editar manualmente todo.

**B) Deploy fragil:**
Produccion no tiene dependency ordering, no hay zero-downtime updates, alertas
se disparan pero nadie las recibe, y si algo se rompe a las 3am nadie se entera.

**C) Observabilidad incompleta:**
HTTP y LLM tracing funcionan, pero DB queries, NATS messages, Redis, y gRPC son
invisibles. No hay business metrics. Logs no tienen user/tenant context.

### Estado actual (auditoria)

| Area | Estado | Nota |
|---|---|---|
| Health checks | Excelente | Todos los servicios tienen /health + healthcheck en compose |
| Restart policies | Excelente | `unless-stopped` everywhere |
| Graceful shutdown | Excelente | 30s timeout, SIGTERM, drain en todos los servicios |
| Backup/restore | Excelente | pg_dump + age encryption + S3 |
| Dependency ordering (prod) | **Malo** | Sin `depends_on: service_healthy` en prod |
| Alert routing | **No existe** | Prometheus alertas definidas pero sin Alertmanager |
| Log shipping | **Parcial** | Loki existe pero no recolecta automaticamente |
| DB/NATS/Redis tracing | **No existe** | Queries y mensajes invisibles |
| Configuracion portable | **Mala** | 15+ hardcodes de dominio |

### Principios

1. **5 variables, 1 archivo** — todo el deploy se configura con un `.env`
2. **Si se cae, se levanta solo** — restart + health checks + autoheal + dependency ordering
3. **Si algo esta mal, te avisa** — alerting real (Alertmanager → Slack/email)
4. **Zero-downtime** — blue-green deploy (diferido post-deadline, requiere rediseño Traefik)
5. **Preflight validation** — errores de config detectados ANTES de levantar

### Fase 1: `.env.example` + template infrastructure

Crear `.env.example` con todas las variables documentadas. Crear script
`gen-config.sh` que genera Traefik/Cloudflare configs desde templates.

**Archivos nuevos:**
- `deploy/.env.example` — template con defaults dev
- `deploy/scripts/gen-config.sh` — genera configs desde templates + `.env`
- `deploy/scripts/detect-host.sh` — auto-detecta WSL2 vs Linux vs Docker Desktop

**El script `gen-config.sh` (fix H5: explicit variable list, fix R2-H4: complete list):**
```bash
#!/bin/bash
set -euo pipefail
source deploy/.env

export SDA_DOMAIN_ESCAPED=$(echo "$SDA_DOMAIN" | sed 's/\./\\\\./g')
export SDA_SMTP_FROM="${SMTP_FROM:-noreply@${SDA_DOMAIN}}"
export SDA_WS_ORIGINS="https://*.${SDA_DOMAIN}"
export SDA_CORS_REGEX="^https://[a-z0-9-]+\\.${SDA_DOMAIN_ESCAPED}\$"

# Only substitute SDA_* variables — prevent replacing Go template syntax in alerts.yml
VARS='$SDA_DOMAIN $SDA_DOMAIN_ESCAPED $SDA_ACME_EMAIL $SDA_TENANT_SLUG
      $SDA_GPU_DEVICES $SDA_ENV $SDA_SMTP_FROM $SDA_WS_ORIGINS $SDA_CORS_REGEX'

envsubst "$VARS" < deploy/traefik/traefik.prod.yml.tmpl > deploy/traefik/traefik.prod.yml
envsubst "$VARS" < deploy/traefik/dynamic/prod.yml.tmpl > deploy/traefik/dynamic/prod.yml
envsubst "$VARS" < deploy/cloudflare/config.yml.tmpl > deploy/cloudflare/config.yml

echo "✓ Configs generated for domain: $SDA_DOMAIN"
```

**Log enrichment (Fase 14 manual, fix H8 — NO usar otelslog):**

Decisión: mantener stdout JSON (probado) + Promtail (Fase 7) para shipping.
Agregar middleware que inyecta `user_id`, `tenant_id`, `trace_id` al slog context.
Descartar otelslog porque reemplaza el log pipeline entero y rompe Promtail.

```go
// pkg/middleware/logging.go
func EnrichLogger(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        logger := slog.Default().With(
            "tenant_id", r.Header.Get("X-Tenant-ID"),
            "user_id", r.Header.Get("X-User-ID"),
            "request_id", middleware.GetReqID(r.Context()),
            "trace_id", trace.SpanFromContext(r.Context()).SpanContext().TraceID().String(),
        )
        ctx := ContextWithLogger(r.Context(), logger)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```
