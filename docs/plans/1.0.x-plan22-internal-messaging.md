## Qué era

Spec original del sistema de mensajería interno. Fue escrito antes de Plans 23/24 y definía la arquitectura completa: Bun WebSocket sidecar, Redis Pub/Sub, SQLite FTS5, canales, DMs, threads, reacciones, typing indicators, presence.

### Por qué fue superseded

El orden de ejecución cambió: Plan 23 → Plan 24 → Plan 22 (polish) → **Plan 25** (messaging real). Este documento fue la spec de diseño. La implementación real se hizo en Plan 25 con ajustes basados en lo aprendido en Plans 23/24.

### Stack definido (usado en Plan 25)

| Componente | Tecnología |
|---|---|
| Real-time | Bun native WebSocket sidecar en :3001 |
| Fan-out | Redis Pub/Sub |
| Presencia | Redis TTL keys |
| Persistencia | SQLite (Drizzle) — `schema/messaging.ts` |
| Búsqueda | SQLite FTS5 |
| Notificaciones | WebSocket + BullMQ |
