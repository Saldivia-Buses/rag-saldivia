## Qué se hizo

Sistema de permisos granular real: roles custom, permisos por feature, usuarios con múltiples roles, y panel admin para gestionarlos.

### Modelo de datos

```
roles (catálogo)              permissions (catálogo)
├── Admin (level 100)         ├── admin.access, users.view, users.manage
├── Manager (level 50)        ├── roles.manage, chat.use, chat.manage_all
├── User (level 10)           ├── collections.read, collections.manage
└── [custom...]               ├── upload.use, templates.manage, settings.manage

role_permissions (N:M)        user_role_assignments (N:M, multi-rol)
```

**Resolución:** unión de permisos de todos los roles del usuario. El level más alto determina rol principal para display.

### Archivos creados/modificados

- **Schema:** `packages/db/src/schema.ts` — tablas `roles`, `permissions`, `role_permissions`, `user_role_assignments`
- **Queries:** `packages/db/src/queries/rbac.ts` — CRUD roles, permisos, resolución
- **Actions:** `apps/web/src/app/actions/roles.ts` — 8 server actions con `authAction`
- **UI:** `apps/web/src/components/admin/AdminRoles.tsx` — CRUD roles con matriz de permisos
- **Middleware:** `apps/web/src/lib/auth/rbac.ts` — actualizado para permisos granulares
- **Proxy:** `apps/web/src/proxy.ts` — RBAC unificado con lib

### Resultado

- 3 roles hardcodeados → sistema extensible de roles custom
- Campo `users.role` → tabla `user_role_assignments` (N:M)
- Panel `/admin/roles` completo con matriz de permisos
- Panel `/admin/users` actualizado para asignar múltiples roles
- Seed con 3 roles default + 11 permisos
