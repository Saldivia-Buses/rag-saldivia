## Contexto

### El problema

El spec (plan 01) define gRPC como el protocolo de comunicacion interna:

> **API inter-servicio:** gRPC (interno) + REST (externo)

Y la arquitectura del WS Hub lo nombra explicitamente:

```
Frontend ──REST/WS──► WebSocket Hub ──gRPC──► Auth / Chat / RAG Service
```

Sin embargo, la realidad es:

- **6 proto files** existen en `proto/` (1187 lineas, 41 RPCs definidos)
- **0 archivos `.pb.go` generados** — ningun codigo Go existe
- **0 `buf.yaml` ni `buf.gen.yaml`** — no hay configuracion de codegen
- **`make proto` referencia `buf generate proto/`** pero buf no esta configurado
- **Toda la comunicacion inter-service es HTTP REST** (agent→search, agent→ingest, agent→notification)
- **El WS Hub rutea via NATS**, no via gRPC como dice el spec
- **El agent executor usa `http.Client`** para llamar a otros servicios

### Por que ahora (y no antes)

1. Plan 07 consolido los paquetes compartidos (`pkg/llm`, `pkg/nats`, `pkg/config`)
2. Plan 08 cierra gaps de seguridad y reliability
3. gRPC es la pieza que falta para cerrar la brecha entre spec y realidad
4. El agent service ya hardcodea HTTP URLs a 3 servicios — migrar a gRPC es el momento

### Que NO resuelve este plan

- **No migra REST a gRPC para el frontend** — REST/WS se mantiene para browser
- **No implementa service mesh** — overkill para el tamano actual
- **No agrega mTLS** — se usa JWT forwarding y Docker network isolation
- **No toca NATS** — NATS sigue siendo el canal para eventos async

## Decision: gRPC scope minimo

### Principio: "gRPC donde hay llamada sincrona entre servicios"

| Patron | Protocolo | Ejemplo |
|---|---|---|
| Frontend → Backend | REST (via Traefik) | Browser → Auth login |
| Frontend → Backend | WebSocket | Browser → WS Hub |
| Servicio → Servicio (sync) | **gRPC** (nuevo) | Agent → Search, WS Hub → Chat |
| Servicio → Servicio (async) | NATS JetStream | Chat → Notification |

### Llamadas inter-service que existen HOY (HTTP, a migrar)

Conteo exacto del codigo:

| Caller | Callee | Como | Archivo |
|---|---|---|---|
| agent | search | HTTP POST `SEARCH_SERVICE_URL/v1/search/query` | `services/agent/cmd/main.go:74` |
| agent | ingest | HTTP POST `INGEST_SERVICE_URL/v1/ingest/upload` | `services/agent/cmd/main.go:77` |
| agent | ingest | HTTP GET `INGEST_SERVICE_URL/v1/ingest/jobs` | `services/agent/cmd/main.go:80` |
| agent | notification | HTTP POST `NOTIFICATION_SERVICE_URL/v1/notifications/send` | `services/agent/cmd/main.go:83` |

**Total: 4 llamadas HTTP inter-service, todas desde agent.**

El WS Hub actualmente NO hace llamadas sync a ningun servicio — solo consume NATS.
Las mutations estan stub (`"mutations not yet implemented"` en `hub.go:177`).

### Proto files existentes vs necesidad real

| Proto | RPCs | Necesita gRPC hoy? | Razon |
|---|---|---|---|
| `auth/v1` | 6 | No | JWT se verifica localmente. Login/logout son REST del frontend. |
| `chat/v1` | 7 | Si (futuro) | WS Hub necesitara llamar Chat via gRPC para mutations. No en este plan. |
| `ingest/v1` | 4 | Si | Agent llama `Upload` y `ListJobs` |
| `notification/v1` | 6 | Parcial | Agent llama 1 RPC. Pero notification es event-driven (NATS). |
| `platform/v1` | 15 | No | Solo lo llama el frontend (REST). Platform admin UI. |
| `rag/v1` | 3 | No (deprecated) | RAG service deprecated. Reemplazado por agent+search. |

### Decision pragmatica

**Fase 1-3:** Infraestructura de codegen + paquete compartido gRPC (aplica a todos).
**Fase 4:** Migrar agent → search a gRPC (la llamada mas frecuente).
**Fase 5:** Migrar agent → ingest a gRPC.
**Fase 6:** Docker networking + security.
**Futuro (no este plan):** WS Hub mutations via gRPC a Chat, agent → notification via gRPC.

## Fases

### Fase 0: Security design

**Por que:** antes de escribir codigo, definir las restricciones de seguridad para
gRPC inter-service. Del Plan 08, los requisitos son claros.

**Decisiones:**

1. **JWT forwarding como gRPC metadata.** El caller pasa el JWT del usuario en
   metadata key `authorization` (formato `Bearer {token}`). El callee verifica
   con la misma clave publica. No hay service-to-service tokens por ahora.

2. **gRPC ports en Docker internal network only.** Los gRPC ports (50051, 50052...)
   NO se exponen a Traefik ni al host. Solo accesibles entre containers en la
   misma Docker network. El frontend nunca ve gRPC.

3. **Auth interceptor unary + stream.** `pkg/grpc/interceptors.go` implementa
   un UnaryServerInterceptor y StreamServerInterceptor que extraen JWT de metadata,
   verifican firma y claims, inyectan tenant context. Equivalente exacto del
   middleware HTTP `pkg/middleware/auth.go`.

4. **No reflection en produccion.** `grpc.reflection.Register(s)` solo en builds
   con `-tags dev`. Produccion no expone reflection.

5. **Request size limits.** `grpc.MaxRecvMsgSize(4 * 1024 * 1024)` — 4MB por
   defecto. Excepcion: Ingest Upload stream (configurable, 100MB max).

6. **Keepalive.** Server: `MaxConnectionIdle=5m, MaxConnectionAge=30m,
   Time=2m, Timeout=20s`. Client: `Time=2m, Timeout=20s, PermitWithoutStream=true`.

**No se escriben archivos.** Esta fase es solo decision. Las decisiones se implementan
en las fases siguientes.

**Esfuerzo:** 0 (decision)

```go
package sdagrpc

import (
    "context"
    "crypto/ed25519"
    "log/slog"
    "strings"

    sdajwt "github.com/Camionerou/rag-saldivia/pkg/jwt"
    sdamw "github.com/Camionerou/rag-saldivia/pkg/middleware"
    "github.com/Camionerou/rag-saldivia/pkg/security"
    "github.com/Camionerou/rag-saldivia/pkg/tenant"
    "google.golang.org/grpc"
    "google.golang.org/grpc/codes"
    "google.golang.org/grpc/metadata"
    "google.golang.org/grpc/status"
)

// InterceptorConfig holds dependencies for the gRPC auth interceptor.
type InterceptorConfig struct {
    PublicKey ed25519.PublicKey
    Blacklist *security.TokenBlacklist // nil = no blacklist
    FailOpen  bool                     // same semantics as HTTP AuthConfig.FailOpen
}

// AuthUnaryInterceptor verifies JWT from gRPC metadata and injects
// tenant context, role, and permissions. Full parity with HTTP middleware.
func AuthUnaryInterceptor(cfg InterceptorConfig) grpc.UnaryServerInterceptor {
    return func(ctx context.Context, req any, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (any, error) {
        if info.FullMethod == "/grpc.health.v1.Health/Check" {
            return handler(ctx, req)
        }
        newCtx, err := extractAndVerifyJWT(ctx, cfg)
        if err != nil {
            return nil, err
        }
        return handler(newCtx, req)
    }
}

// AuthStreamInterceptor is the streaming equivalent.
func AuthStreamInterceptor(cfg InterceptorConfig) grpc.StreamServerInterceptor {
    return func(srv any, ss grpc.ServerStream, info *grpc.StreamServerInfo, handler grpc.StreamHandler) error {
        if info.FullMethod == "/grpc.health.v1.Health/Check" {
            return handler(srv, ss)
        }
        newCtx, err := extractAndVerifyJWT(ss.Context(), cfg)
        if err != nil {
            return err
        }
        return handler(srv, &wrappedStream{ss, newCtx})
    }
}

func extractAndVerifyJWT(ctx context.Context, cfg InterceptorConfig) (context.Context, error) {
    md, ok := metadata.FromIncomingContext(ctx)
    if !ok {
        return nil, status.Error(codes.Unauthenticated, "missing metadata")
    }
    vals := md.Get("authorization")
    if len(vals) == 0 {
        return nil, status.Error(codes.Unauthenticated, "missing authorization")
    }
    token := vals[0]
    if strings.HasPrefix(token, "Bearer ") {
        token = token[7:]
    }

    // Verify signature + claims (B4 fix: correct argument order)
    claims, err := sdajwt.Verify(cfg.PublicKey, token)
    if err != nil {
        slog.Warn("grpc auth failed", "error", err)
        return nil, status.Error(codes.Unauthenticated, "invalid token")
    }

    // Blacklist check (D2 fix: parity with HTTP middleware)
    if cfg.Blacklist != nil {
        if claims.ID == "" {
            return nil, status.Error(codes.Unauthenticated, "invalid token")
        }
        revoked, err := cfg.Blacklist.IsRevoked(ctx, claims.ID)
        if err != nil {
            slog.Error("grpc blacklist check failed", "error", err)
            if !cfg.FailOpen {
                return nil, status.Error(codes.Unavailable, "auth check unavailable")
            }
        } else if revoked {
            return nil, status.Error(codes.Unauthenticated, "token revoked")
        }
    }

    // Reject MFA-pending tokens (D3 fix: parity with HTTP middleware)
    if claims.Role == "mfa_pending" {
        return nil, status.Error(codes.Unauthenticated, "mfa verification required")
    }

    // Inject tenant, role, permissions into context (D1 fix: full parity)
    ctx = tenant.WithInfo(ctx, tenant.Info{
        ID:   claims.TenantID,
        Slug: claims.Slug,
    })
    ctx = sdamw.WithRole(ctx, claims.Role)
    ctx = sdamw.WithPermissions(ctx, claims.Permissions)

    return ctx, nil
}

// JWTFromIncomingContext extracts the bearer token from incoming gRPC metadata.
// Used for gRPC-to-gRPC forwarding (D7 fix).
func JWTFromIncomingContext(ctx context.Context) (string, bool) {
    md, ok := metadata.FromIncomingContext(ctx)
    if !ok {
        return "", false
    }
    vals := md.Get("authorization")
    if len(vals) == 0 {
        return "", false
    }
    token := vals[0]
    if strings.HasPrefix(token, "Bearer ") {
        return token[7:], true
    }
    return token, true
}

// wrappedStream overrides Context() for stream interceptors.
type wrappedStream struct {
    grpc.ServerStream
    ctx context.Context
}
func (w *wrappedStream) Context() context.Context { return w.ctx }
```

**`pkg/grpc/server.go`:**

```go
package sdagrpc

import (
    "crypto/ed25519"
    "time"

    "google.golang.org/grpc"
    "google.golang.org/grpc/keepalive"
)

// DefaultMaxRecvMsgSize is 4MB (sufficient for most RPCs).
const DefaultMaxRecvMsgSize = 4 * 1024 * 1024

// NewServer creates a gRPC server with standard SDA interceptors and limits.
// IMPORTANT: additional opts must use ChainUnaryInterceptor, NOT UnaryInterceptor
// (the latter would silently replace the auth interceptor).
func NewServer(cfg InterceptorConfig, opts ...grpc.ServerOption) *grpc.Server {
    defaults := []grpc.ServerOption{
        grpc.MaxRecvMsgSize(DefaultMaxRecvMsgSize),
        grpc.ChainUnaryInterceptor(AuthUnaryInterceptor(cfg)),
        grpc.ChainStreamInterceptor(AuthStreamInterceptor(cfg)),
        grpc.KeepaliveParams(keepalive.ServerParameters{
            MaxConnectionIdle: 5 * time.Minute,
            MaxConnectionAge:  30 * time.Minute,
            Time:              2 * time.Minute,
            Timeout:           20 * time.Second,
        }),
    }
    return grpc.NewServer(append(defaults, opts...)...)
}
```

**`pkg/grpc/client.go`:**

```go
package sdagrpc

import (
    "context"
    "time"

    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
    "google.golang.org/grpc/keepalive"
    "google.golang.org/grpc/metadata"
)

// Dial creates a gRPC client connection with standard SDA defaults.
// Uses insecure transport (internal Docker network only — never exposed externally).
// Uses grpc.NewClient (lazy connection — does not fail on unreachable target).
func Dial(target string, opts ...grpc.DialOption) (*grpc.ClientConn, error) {
    defaults := []grpc.DialOption{
        grpc.WithTransportCredentials(insecure.NewCredentials()),
        grpc.WithKeepaliveParams(keepalive.ClientParameters{
            Time:                2 * time.Minute,
            Timeout:             20 * time.Second,
            PermitWithoutStream: true,
        }),
        grpc.WithDefaultCallOptions(
            grpc.MaxCallRecvMsgSize(DefaultMaxRecvMsgSize),
        ),
    }
    return grpc.NewClient(target, append(defaults, opts...)...)
}

// ForwardJWT creates outgoing metadata with the JWT from the caller.
// Used by services that proxy requests to other services.
func ForwardJWT(ctx context.Context, jwt string) context.Context {
    return metadata.AppendToOutgoingContext(ctx, "authorization", "Bearer "+jwt)
}
```

```go
package handler

import (
    "context"

    searchv1 "github.com/Camionerou/rag-saldivia/gen/go/search/v1"
    "github.com/Camionerou/rag-saldivia/services/search/internal/service"
    "google.golang.org/grpc/codes"
    "google.golang.org/grpc/status"
)

// GRPCHandler implements the SearchService gRPC server.
type GRPCHandler struct {
    searchv1.UnimplementedSearchServiceServer
    svc *service.Search
}

func NewGRPC(svc *service.Search) *GRPCHandler {
    return &GRPCHandler{svc: svc}
}

func (h *GRPCHandler) Query(ctx context.Context, req *searchv1.SearchRequest) (*searchv1.SearchResponse, error) {
    if req.Query == "" {
        return nil, status.Error(codes.InvalidArgument, "query is required")
    }

    collectionID := ""
    if req.CollectionId != nil {
        collectionID = *req.CollectionId
    }

    // B5 fix: correct method name is SearchDocuments, not Query
    result, err := h.svc.SearchDocuments(ctx, req.Query, collectionID, int(req.MaxNodes))
    if err != nil {
        // D6 fix: generic error message, log real error
        slog.Error("grpc search failed", "error", err)
        return nil, status.Error(codes.Internal, "search failed")
    }
    return toProto(result), nil
}

// B7 fix: toProto converts service result to proto response
func toProto(r *service.SearchResult) *searchv1.SearchResponse {
    resp := &searchv1.SearchResponse{
        Query:      r.Query,
        DurationMs: int32(r.DurationMS),
    }
    for _, s := range r.Selections {
        pages := make([]int32, len(s.Pages))
        for i, p := range s.Pages { pages[i] = int32(p) }
        resp.Selections = append(resp.Selections, &searchv1.Selection{
            Document:   s.Document,
            DocumentId: s.DocumentID,
            NodeIds:    s.NodeIDs,
            Sections:   s.Sections,
            Pages:      pages,
            Text:       s.Text,
            Tables:     s.Tables,
            Images:     s.Images,
        })
    }
    return resp
}
```

```go
// GRPCSearchClient wraps the search gRPC client for tool execution.
type GRPCSearchClient struct {
    client searchv1.SearchServiceClient
    conn   *grpc.ClientConn
}

func NewGRPCSearchClient(target string) (*GRPCSearchClient, error) {
    conn, err := sdagrpc.Dial(target)
    if err != nil {
        return nil, err
    }
    return &GRPCSearchClient{
        client: searchv1.NewSearchServiceClient(conn),
        conn:   conn,
    }, nil
}

func (c *GRPCSearchClient) Close() error {
    return c.conn.Close()
}

// Execute runs the search_documents tool via gRPC instead of HTTP.
func (c *GRPCSearchClient) Execute(ctx context.Context, jwt string, params json.RawMessage) (*Result, error) {
    // Forward JWT via gRPC metadata
    ctx = sdagrpc.ForwardJWT(ctx, jwt)

    var p struct {
        Query        string `json:"query"`
        CollectionID string `json:"collection_id"`
        MaxNodes     int    `json:"max_nodes"`
    }
    if err := json.Unmarshal(params, &p); err != nil {
        return &Result{Status: "error", Error: "invalid params"}, nil
    }

    resp, err := c.client.Query(ctx, &searchv1.SearchRequest{
        Query:        p.Query,
        CollectionId: &p.CollectionID,
        MaxNodes:     int32(p.MaxNodes),
    })
    if err != nil {
        return &Result{Status: "error", Error: err.Error()}, nil
    }

    data, _ := json.Marshal(resp)
    return &Result{Status: "success", Data: data}, nil
}
```

```go
// GRPCExecutor is implemented by gRPC tool clients.
type GRPCExecutor interface {

}

// RegisterGRPC registers a gRPC executor for a tool name.
// When Execute is called for this tool, gRPC is used instead of HTTP.
func (e *Executor) RegisterGRPC(toolName string, exec GRPCExecutor) {
    if e.grpcTools == nil {
        e.grpcTools = make(map[string]GRPCExecutor)
    }
    e.grpcTools[toolName] = exec
}
```

## Fuera de scope (futuro, no este plan)

| Feature | Por que no ahora | Cuando |
|---|---|---|
| WS Hub → Chat via gRPC (mutations) | Mutations estan stub, no implementadas aun | Cuando se implementen mutations |
| Agent → Notification via gRPC | Notification es event-driven (NATS), 1 sola llamada sync | Cuando haya mas RPCs |
| Auth gRPC server | JWT se verifica localmente, no necesita round-trip | Nunca (o si agrega RPCs nuevos) |
| Platform gRPC server | Solo lo llama el frontend (REST) | Si otro servicio necesita config |
| mTLS entre servicios | Docker network isolation es suficiente hoy | Cuando haya multi-host |
| gRPC-Web / grpc-gateway | El frontend usa REST, no necesita gRPC | Si algun dia se quiere |
| Load balancing gRPC | 1 instancia por servicio hoy | Cuando se escale horizontalmente |
| Service discovery (Consul/etcd) | DNS de Docker es suficiente | Multi-host |
