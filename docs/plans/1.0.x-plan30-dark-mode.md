## Fases

### Fase 0: Decisión de diseño — `--accent-fg` en dark mode

**Problema:** En dark mode, `--accent: #60a5fa` (azul claro) y `--accent-fg: #0c1929` (navy oscuro).
Pero 7+ componentes usan `bg-accent text-white` en vez de `bg-accent text-accent-fg`.
Si reemplazamos `text-white` mecánicamente, los botones quedan ilegibles (texto oscuro sobre azul claro).

**Hallazgos del review:**
| Archivo | Línea | Código |
|---------|-------|--------|
| `ChannelList.tsx` | 129 | `bg-accent text-white` |
| `MessageComposer.tsx` | 134 | `bg-accent text-white hover:bg-accent/90` |
| `ThreadPanel.tsx` | 177 | `bg-accent text-white hover:bg-accent/90` |
| `UnreadBadge.tsx` | 11 | `bg-accent text-white` |
| `MessageActions.tsx` | 92 | `bg-accent text-white hover:bg-accent/90` |
| `MessageActions.tsx` | 118 | `bg-destructive text-white hover:bg-destructive/90` |
| `button.tsx` | 14 | `bg-destructive text-white` |

**Decisión a tomar:** Cambiar `--accent-fg` en dark mode a `#ffffff` (blanco sobre azul claro = legible),
o mantener `text-white` hardcodeado (funciona pero no es semántico).

**Recomendación:** Cambiar el token. `--accent-fg: #ffffff` en dark mode es correcto porque
el accent es claro. Después reemplazar `text-white` por `text-accent-fg` en los 7+ lugares.

**Commit:** `style(ui): fix --accent-fg dark mode token — plan30 f0`

### Fase 1: Fix ArtifactPanel — Mermaid theme + hardcoded colors

**Problemas encontrados:**
1. **Mermaid theme** (líneas 152-172): 15+ hex hardcodeados mezclando tokens light y dark.
   `primaryColor: "#2563eb"` (light accent), `textColor: "#f5f4f0"` (dark fg), etc.
   Esto es un bug visible: los diagramas siempre se ven como un híbrido light/dark.
   **Fix:** Leer theme actual con `useTheme()` (ya importado) y cambiar el theme config según mode.

2. **`bg-white`** (línea 223): HtmlPreview iframe con fondo blanco hardcodeado.
   **Fix:** `bg-bg` (usa token).

3. **TYPE_CONFIG** (líneas 241-250): 4 hex para colores de tipo (HTML=#e34c26, SVG=#ffb13b, etc.).
   Estos son "brand colors" de lenguajes — aceptable dejarlos fijos. Solo documentar la decisión.

**Verificación:**
- [ ] `bunx tsc --noEmit` → 0 errors
- [ ] Mermaid diagrams se ven bien en light Y dark

**Commit:** `style(chat): fix ArtifactPanel dark mode (Mermaid theme + bg-white) — plan30 f1`
