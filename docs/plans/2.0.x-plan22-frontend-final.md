# Plan 22 — Frontend Final: TanStack Query, Error Handling, UI Polish

> **Prerequisito:** Plan 17 + 18 + 19 + 20 completados (todas las paginas ERP + AFIP existen)
> **Fecha:** 2026-04-10
> **Estado:** Draft — pendiente aprobacion
> **Version target:** 2.0.11
> **Sprint:** think > plan > execute > review > test > ship (por fase)
> **Servicios afectados:** `apps/web/`
> **Estimacion:** 6 fases, ~6-8 dias
> **Migrations:** ninguna (frontend only)

## Contexto

Plan 17 implemento 16+ paginas ERP en el frontend. Todas funcionan, pero
tienen deuda tecnica sistematica:

1. **useState/useEffect manual** en todas las paginas — no hay cache, no hay
   retry, no hay deduplication, no hay optimistic updates
2. **Zero error handling en mutations** — si un POST falla, el usuario no ve
   nada. `console.error` no cuenta.
3. **10+ tabs del spec sin implementar** — las paginas muestran los tabs
   principales pero faltan los secundarios
4. **Create flows faltantes** — contable, ventas, facturacion son read-only,
   no se puede crear desde el frontend
5. **Search sin debounce** — cada keystroke hace un API call
6. **Helpers duplicados** — `fmtMoney` esta definido en 8+ archivos
7. **Tildes inconsistentes** — mezcla de "Contabilización" y "Contabilizacion"
8. **Pagination incompleta** — muchas paginas cargan todo sin paginar

**Este plan convierte el frontend ERP de "funciona" a "produccion-ready".**

## Principios

1. **TanStack Query es el unico patron de data fetching.** Cero useState para server state.
2. **Toda mutation tiene feedback.** Toast de exito/error. Loading state en botones.
3. **Helpers se importan, no se copian.** Un archivo, un import.
4. **Mobile-first no, pero responsive si.** Tablas deben ser usables en tablet.

#### format.ts

```typescript
const moneyFormatter = new Intl.NumberFormat("es-AR", {
  style: "currency", currency: "ARS", maximumFractionDigits: 0,
});
const numberFormatter = new Intl.NumberFormat("es-AR", {
  maximumFractionDigits: 2,
});
const dateFormatter = new Intl.DateTimeFormat("es-AR", {
  day: "2-digit", month: "2-digit", year: "numeric",
});

export function fmtMoney(n: number | null | undefined): string {
  if (n == null || n === 0) return "\u2014";
  return moneyFormatter.format(n);
}

export function fmtDate(d: string | null | undefined): string {
  if (!d) return "\u2014";
  return dateFormatter.format(new Date(d));
}

export function fmtNumber(n: number | null | undefined, decimals = 2): string {
  if (n == null) return "\u2014";
  return numberFormatter.format(n);
}

export function fmtPercent(n: number | null | undefined): string {
  if (n == null) return "\u2014";
  return `${(n * 100).toFixed(1)}%`;
}

export function parseDecimal(s: string): number {
  // Handles "1.234,56" (es-AR format) → 1234.56
  return parseFloat(s.replace(/\./g, "").replace(",", "."));
}
```

### Mutations con feedback (ejemplo crear asiento)

```typescript
function CreateEntryDialog({ onClose }: { onClose: () => void }) {
  const queryClient = useQueryClient();
  const mutation = useMutation({
    mutationFn: (data: CreateEntryInput) => api.post("/v1/erp/accounting/entries", data),
    onSuccess: (result) => {
      toast.success(`Asiento ${result.entry.number} creado`);
      queryClient.invalidateQueries({ queryKey: erpKeys.entries() });
      queryClient.invalidateQueries({ queryKey: erpKeys.balance() });
      onClose();
    },
    onError: (err) => {
      toast.error("Error al crear asiento", {
        description: err instanceof Error ? err.message : "Verificar que el asiento balancea",
      });
    },
  });

  return (
    <Dialog>
      {/* form fields */}
      <Button onClick={() => mutation.mutate(formData)} disabled={mutation.isPending}>
        {mutation.isPending ? "Creando..." : "Crear asiento"}
      </Button>
    </Dialog>
  );
}
```

### Search con `useDeferredValue` (React 19)

**Antes:** API call on every keystroke.

**Despues:** Mismo patron que en fase 0 — `useDeferredValue` de React 19.
Ver seccion "Debounce approach" en fase 0 para el ejemplo completo.
Aplicar a todas las paginas que tengan inputs de busqueda.

#### Debounce approach

El proyecto usa React 19.2.4 que incluye `useDeferredValue` built-in.
Para search inputs, preferir `useDeferredValue` sobre un hook custom:

```typescript
import { useDeferredValue, useState } from "react";

function ArticleSearch() {
  const [search, setSearch] = useState("");
  const deferredSearch = useDeferredValue(search);

  const { data } = useQuery({
    queryKey: erpKeys.stockArticles({ search: deferredSearch }),
    queryFn: () => api.get(`/v1/erp/stock/articles?search=${deferredSearch}`),
    enabled: deferredSearch.length >= 2,
  });

  return <Input value={search} onChange={e => setSearch(e.target.value)} placeholder="Buscar artículos..." />;
}
```

`useDeferredValue` es mejor que un debounce custom porque:
- Built-in de React 19, zero dependencies
- No introduce delay artificial — React decide cuando actualizar
- Mantiene la UI responsive mostrando el valor anterior mientras calcula
- Se integra nativamente con Suspense y transitions

### Patron de create dialog

Todos los create dialogs siguen el mismo patron:

```typescript
function CreateDialog<T>({
  title, fields, mutationFn, queryKey, onClose
}: {
  title: string;
  fields: FieldConfig[];
  mutationFn: (data: T) => Promise<any>;
  queryKey: readonly unknown[];
  onClose: () => void;
}) {
  const queryClient = useQueryClient();
  const mutation = useMutation({
    mutationFn,
    onSuccess: () => {
      toast.success(`${title} creado exitosamente`);
      queryClient.invalidateQueries({ queryKey });
      onClose();
    },
    onError: (err) => toast.error(`Error al crear ${title.toLowerCase()}`, {
      description: err instanceof Error ? err.message : undefined,
    }),
  });
  // ... form render using fields config
}
```

```typescript
// apps/web/src/components/erp/Pagination.tsx
function ERPPagination({
  page, pageSize, total, onPageChange
}: {
  page: number; pageSize: number; total: number;
  onPageChange: (page: number) => void;
}) {
  const totalPages = Math.ceil(total / pageSize);
  return (
    <div className="flex items-center justify-between py-4">
      <span className="text-sm text-muted-foreground">
        {total} registros — Página {page} de {totalPages}
      </span>
      <div className="flex gap-2">
        <Button variant="outline" size="sm" disabled={page <= 1}
          onClick={() => onPageChange(page - 1)}>Anterior</Button>
        <Button variant="outline" size="sm" disabled={page >= totalPages}
          onClick={() => onPageChange(page + 1)}>Siguiente</Button>
      </div>
    </div>
  );
}
```
