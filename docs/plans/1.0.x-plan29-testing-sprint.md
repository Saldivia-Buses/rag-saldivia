RoleCard, RoleForm, CreateUserForm, PasswordResetCell) también se testean.

ChatEmptyState del scope. Agregar ChatLayout.

singleton en `@/lib/ws/client`) y server actions (`@/app/actions/messaging`). Ninguno
está mockeado en el test setup actual. Fase 0 resuelve esto.

## Fases

### Fase 0: Setup de mocks para messaging

**Archivo:** `apps/web/src/lib/component-test-setup.ts`

**Mocks necesarios:**
```typescript
// WebSocket client singleton — usado por useMessaging, useTyping
mock.module("@/lib/ws/client", () => ({
  wsClient: {
    send: () => {},
    subscribe: () => () => {},
    connect: () => {},
    disconnect: () => {},
    isConnected: () => false,
  },
}))

// Server actions de messaging — 5+ componentes las importan
mock.module("@/app/actions/messaging", () => ({
  actionSendMessage: async () => ({ data: { id: "mock-msg" } }),
  actionEditMessage: async () => ({ data: true }),
  actionDeleteMessage: async () => ({ data: true }),
  actionCreateChannel: async () => ({ data: { id: 1 } }),
  actionPinMessage: async () => ({ data: true }),
  actionUnpinMessage: async () => ({ data: true }),
  actionAddReaction: async () => ({ data: true }),
  actionRemoveReaction: async () => ({ data: true }),
}))
```

**También verificar:**
- `react-hotkeys-hook` funciona en happy-dom (si no, mock)
- `navigator.clipboard` disponible (si no, mock)
- `cmdk` renderiza en happy-dom (si no, mock `@/components/ui/command`)
- `window.SpeechRecognition` para VoiceInput (happy-dom no lo tiene — componente tiene fallback `if (!supported) return null`)

**Verificación:**
- [ ] Tests existentes siguen pasando (`bun run test:components`)
- [ ] Un smoke test importando useMessaging con el mock no crashea

**Commit:** `test(setup): add wsClient + messaging action mocks for component tests — plan29 f0`
