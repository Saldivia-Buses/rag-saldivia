## Contexto

Auditoria completa del 2026-04-03 (3 auditorias en paralelo: security auditor,
backend explorer, frontend+infra explorer). 28 hallazgos de seguridad
(4 criticos, 8 altos, 8 medios, 8 bajos). Sistema al ~40% del spec. La base es
solida (8 microservicios funcionales, frontend con auth/chat/WS, tenant isolation
estructural, observabilidad LGTM+), pero hay gaps criticos en seguridad que
bloquean produccion:

- RBAC sin enforcement (el JWT tiene `role` pero ningun handler lo verifica)
- MFA con columnas en DB pero zero codigo Go
- Audit log solo registra login exitoso
- JWT usa HS256 (simetrico) — spec dice clave publica compartida (asimetrico)
- DB credentials de tenants en plaintext en Platform DB
- No existe docker-compose.prod.yml
- Sin security headers, sin rate limiting per-service
- Deploy pipeline roto

**Prioridad 1: seguridad. Prioridad 2: infra de produccion.**

La biblia dice: "La seguridad no es un tradeoff. Es una restriccion."

**Servicio:** pkg/jwt, todos los que consumen JWT
**Archivos:**
- `pkg/jwt/jwt.go` — rewrite para Ed25519 (sign con private key, verify con public key)
- `pkg/jwt/jwt_test.go` — tests actualizados
- `pkg/middleware/auth.go` — cambiar Verify a usar public key
- `services/auth/cmd/main.go` — cargar private + public key
- `services/chat/cmd/main.go` — cargar solo public key

- `services/ingest/cmd/main.go` — cargar solo public key
- `services/ws/cmd/main.go` — cargar solo public key
- `services/platform/cmd/main.go` — cambiar a public key verify
- `deploy/docker-compose.dev.yml` — agregar JWT_PRIVATE_KEY / JWT_PUBLIC_KEY env vars

**Cambios:**

`pkg/jwt/jwt.go`:
- Eliminar `SigningMethodHS256`, reemplazar con `SigningMethodEdDSA` (Ed25519)
- `Config` cambia: `Secret string` -> `PrivateKey ed25519.PrivateKey` + `PublicKey ed25519.PublicKey`
- `CreateAccess` y `CreateRefresh` firman con private key
- `Verify` valida con public key
- Agregar helpers: `LoadPrivateKey(path string)`, `LoadPublicKey(path string)` para
  cargar keys PEM, y `ParsePrivateKeyEnv(b64 string)`, `ParsePublicKeyEnv(b64 string)`
  para cargar desde env vars (base64-encoded PEM)
- `DefaultConfig` recibe `privateKey, publicKey` en vez de `secret`
- Mantener backward-compatible `VerifyHS256(secret, token)` para migration period

`pkg/middleware/auth.go`:
- Cambiar de `Auth(jwtSecret string)` a `Auth(publicKey ed25519.PublicKey)`
- Verify usa solo public key — no necesita el secret

Cada servicio excepto auth solo necesita la public key. Auth necesita ambas
(private para firmar, public para verificar refresh tokens).

**Env vars nuevos:**

| Var | Required | Default | Quien lo usa |
|-----|----------|---------|-------------|
| JWT_PRIVATE_KEY | solo auth | — | auth service (firmar tokens) |
| JWT_PUBLIC_KEY | todos | — | todos los servicios (verificar tokens) |

En dev, se generan automaticamente al levantar con `make dev`. En prod, se
configuran como Docker secrets.

**Script para generar keys (agregar a deploy/scripts/):**
```bash
# deploy/scripts/gen-jwt-keys.sh
openssl genpkey -algorithm Ed25519 -out jwt-private.pem
openssl pkey -in jwt-private.pem -pubout -out jwt-public.pem
echo "Keys generated: jwt-private.pem, jwt-public.pem"
echo "Base64 for env vars:"
echo "JWT_PRIVATE_KEY=$(base64 -w0 jwt-private.pem)"
echo "JWT_PUBLIC_KEY=$(base64 -w0 jwt-public.pem)"
```

```go
// RequirePermission returns middleware that checks if the user has the given
// permission. The permission must be present in the X-User-Permissions header
// (comma-separated list set by the auth middleware after loading from DB).
//
// Usage:
//   r.With(middleware.RequirePermission("chat.read")).Get("/sessions", h.ListSessions)
//   r.With(middleware.RequirePermission("ingest.write")).Post("/upload", h.Upload)
func RequirePermission(permission string) func(http.Handler) http.Handler
```

Estrategia de carga de permisos — dos opciones:

**Opcion A (elegida): Permissions en el JWT claim.**
Agregar `Permissions []string` al JWT Claims struct. Se setean al login/refresh.
El middleware RBAC lee del claim sin DB roundtrip. Downside: si cambian los
permisos, el usuario debe esperar al refresh (max 15 min). Aceptable para v1.

Esto significa cambiar `pkg/jwt/jwt.go` Claims:
```go
type Claims struct {
    gojwt.RegisteredClaims
    UserID      string   `json:"uid"`
    Email       string   `json:"email"`
    Name        string   `json:"name"`
    TenantID    string   `json:"tid"`
    Slug        string   `json:"slug"`
    Role        string   `json:"role"`
    Permissions []string `json:"perms,omitempty"` // NUEVO
}
```

`services/auth/internal/service/auth.go`:
- En `Login()` y `Refresh()`, despues de `getPrimaryRole()`, agregar `getPermissions(ctx, userID) ([]string, error)`
- La query: `SELECT p.id FROM permissions p JOIN role_permissions rp ON rp.permission_id = p.id JOIN user_roles ur ON ur.role_id = rp.role_id WHERE ur.user_id = $1`
- Setear `claims.Permissions = permissions`

`pkg/middleware/auth.go`:
- Agregar `X-User-Permissions` header con permissions comma-separated
- O mejor: poner en context via `context.WithValue`

`pkg/middleware/rbac.go`:
- Lee permissions del context (seteado por auth middleware)
- Si la permission requerida no esta en la lista → 403 Forbidden
- Admin role bypasea todo (por definicion tiene todos los permisos)

**Cambios:**

Flujo MFA:
```
1. Usuario hace login normal (email + password)
2. Si mfa_enabled = true para ese usuario:
   a. Login devuelve 200 con {mfa_required: true, mfa_token: "temp-jwt"}
      (el mfa_token es un JWT de vida corta, 5 min, con claim mfa_pending=true)
   b. NO devuelve access_token ni refresh_token
   c. Usuario envia TOTP code: POST /v1/auth/mfa/verify {mfa_token, code}
   d. Si code es valido → devuelve access_token + refresh_token normales
   e. Si code es invalido → 401
3. Si mfa_enabled = false → login normal, devuelve tokens directamente
```

`services/auth/internal/service/mfa.go`:
```go
// Package service — MFA (TOTP) functionality

// EnableMFA generates a TOTP secret for the user, stores it encrypted,
// and returns the provisioning URI for QR code generation.
func (a *Auth) EnableMFA(ctx context.Context, userID string) (secret, uri string, err error)

// VerifyMFA validates a TOTP code against the user's stored secret.
func (a *Auth) VerifyMFA(ctx context.Context, userID, code string) error

// DisableMFA removes MFA for the user (requires valid TOTP code as confirmation).
func (a *Auth) DisableMFA(ctx context.Context, userID, code string) error

// CheckMFARequired returns true if the user has MFA enabled.
func (a *Auth) CheckMFARequired(ctx context.Context, userID string) (bool, error)
```

Usa library `github.com/pquerna/otp/totp` (MIT, standard Go TOTP).

El `mfa_secret` se almacena en la columna `users.mfa_secret` que ya existe.
Si Fase 10 (pkg/crypto) ya esta implementada, encriptar con AES-256-GCM antes
de guardar. Si no, almacenar como texto plano temporalmente (la DB ya esta
aislada por tenant) y encriptar en Fase 10 como parte de la migracion.

```go
// SecureHeaders adds security headers to every response.
// These headers protect against common web attacks (clickjacking, MIME sniffing,
// XSS, etc.) and signal to browsers how to handle the content securely.
func SecureHeaders() func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            w.Header().Set("X-Content-Type-Options", "nosniff")
            w.Header().Set("X-Frame-Options", "DENY")
            w.Header().Set("X-XSS-Protection", "0") // disabled, CSP is better
            w.Header().Set("Referrer-Policy", "strict-origin-when-cross-origin")
            w.Header().Set("Permissions-Policy", "camera=(), microphone=(), geolocation=()")
            // HSTS: only set when behind TLS (Traefik/Cloudflare)
            // In dev (HTTP), don't set HSTS to avoid browser lockout
            if r.TLS != nil || r.Header.Get("X-Forwarded-Proto") == "https" {
                w.Header().Set("Strict-Transport-Security", "max-age=31536000; includeSubDomains")
            }
            next.ServeHTTP(w, r)
        })
    }
}
```

CSP queda fuera — lo define el frontend (Next.js) o Traefik, no los servicios API.
