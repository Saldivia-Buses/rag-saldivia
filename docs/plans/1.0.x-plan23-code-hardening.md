> **Branch:** 1.0.x
> **Prerequisito:** Plan 18 completado

> **Orden:** Plan 23 -> Plan 24 (Collections) -> Plan 22 (Messaging)

---

## Contexto

Auditoria profunda del codigo actual + investigacion de best practices 2025-2026
revelo issues concretas y oportunidades de mejora. El codigo es "above average"
(B+ general) pero tiene:

- **Vulnerabilidades de seguridad reales** (XSS via SVG/Mermaid sin sanitizar)
- **Performance bottlenecks** (sync bcrypt, Shiki recreation, unbounded caches)
- **Accesibilidad incompleta** (sin aria-live en chat)
- **Patterns outdated** vs lo que la industria usa en 2026
- **Zero tests** en server actions, API routes, y middleware

Este plan arregla todo esto. Mejora la base sobre la que se construyen Plans 24 y 22.

### Fase 1: Seguridad — XSS, JWT, RBAC

**Archivos:**
- `apps/web/src/components/chat/ArtifactPanel.tsx`
- `apps/web/src/lib/auth/jwt.ts`
- `apps/web/src/proxy.ts`
- `apps/web/src/lib/auth/rbac.ts`
- `apps/web/package.json` (agregar `dompurify` + `@types/dompurify`)

**Cambios:**

1. **XSS en ArtifactPanel (CRITICO):**
   - Instalar `dompurify`
   - Sanitizar output de Mermaid (linea 177) antes de `dangerouslySetInnerHTML`:
     ```typescript
     import DOMPurify from "dompurify"
     const clean = DOMPurify.sanitize(svgOutput, { USE_PROFILES: { svg: true } })
     ```
   - Sanitizar SVG preview (linea 211) — mismo patron
   - Shiki output (linea 68) es trusted (generado por la libreria) — agregar comentario

2. **Cookie Max-Age desincronizada con JWT:**
   - Derivar `maxAge` del mismo `JWT_EXPIRY` que usa `getExpiry()`:
     ```typescript
     function parseExpiry(str: string): number {
       const match = str.match(/^(\d+)(h|d|m)$/)
       if (!match) return 86400 // default 24h
       const [, n, unit] = match
       const multipliers = { m: 60, h: 3600, d: 86400 }
       return Number(n) * multipliers[unit as keyof typeof multipliers]
     }
     ```

3. **RBAC duplicada en proxy.ts:**
   - Exportar `ADMIN_ROUTES` y `canAccessRoute()` desde `lib/auth/rbac.ts`
   - Importar en `proxy.ts` en vez de duplicar la logica inline

**Verificacion:**
- [ ] `bunx tsc --noEmit` -> 0 errors
- [ ] `bun run test` -> green
- [ ] SVG con `<script>` tag es strippeado por DOMPurify
- [ ] Cookie expira al mismo tiempo que el JWT
- [ ] RBAC funciona identico post-refactor

**Commit:** `fix(security): sanitize SVG/Mermaid XSS, sync cookie expiry, unify RBAC — plan23 f1`

### Fase 2b: Performance — Async Bcrypt, Caches, Parallel Actions

**Archivos:**
- `packages/db/src/queries/users.ts`
- `apps/web/src/components/chat/ArtifactPanel.tsx`
- `apps/web/src/components/chat/ChatInterface.tsx`
- `apps/web/src/app/actions/chat.ts`

**Cambios:**

1. **Async bcrypt:**
   - Cambiar `hashSync`/`compareSync` por `hash`/`compare` en `users.ts` (lineas 50, 71, 118)
   - Callers ya son async, solo cambiar las funciones internas

2. **Shiki singleton + LRU cache:**
   - Crear highlighter una sola vez: `let highlighterPromise: Promise<Highlighter>`
   - Reemplazar `hlCache: Map` con Map + eviction simple:
     ```typescript
     function lruSet(map: Map<string, string>, key: string, val: string, max = 100) {
       if (map.size >= max) map.delete(map.keys().next().value!)
       map.set(key, val)
     }
     ```

3. **Promise.all en ChatInterface.onFinish:**
   - Las dos llamadas a `actionAddMessage` (user + assistant) son independientes
   - Cambiar de secuencial a `Promise.all([...])` — ~2x mas rapido

4. **Batch insert en actionForkSession:**
   - Cambiar loop `for...of` + `await insert` individual por un solo `db.insert().values([...])`

**Verificacion:**
- [ ] `bunx tsc --noEmit` -> 0 errors
- [ ] `bun run test` -> green
- [ ] Login funciona con async bcrypt
- [ ] Shiki no re-crea highlighter entre code blocks
- [ ] Fork session es mas rapido con batch insert

**Commit:** `perf(web): async bcrypt, shiki singleton, parallel actions — plan23 f2b`
