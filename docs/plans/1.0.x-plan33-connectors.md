# Plan 33 — Conectores Externos

> **Branch:** 1.0.x
> **Prerequisito:** Plan 26 (BullMQ + Redis estable), Plan 24 (admin system)
> **Sprint:** think -> plan -> execute -> review -> test -> ship
> **Intensity:** thorough

## Contexto

Saldivia RAG tiene 0 conectores automaticos. Glean tiene 100+, Onyx tiene 40+.
La ingesta hoy es 100% manual (upload via API o interfaz). Para los usuarios
target (Enzo, su papa, su tio, y luego la empresa), la promesa del producto es
"centralizar informacion" -- y eso requiere que el sistema vaya a buscar documentos
automaticamente desde las fuentes donde la empresa ya trabaja.

Este plan implementa un sistema de plugins de conectores con 4 conectores
iniciales que cubren el ~80% de los casos de uso empresariales: Google Drive,
SharePoint/OneDrive, Confluence, y Web Crawler.

### Infraestructura existente (ya implementada)

La DB y queries ya existen:

- **Schema:** `packages/db/src/schema/core.ts` lineas 224-243 -- tabla `external_sources` con id, userId, provider, name, credentials (JSON), collectionDest, schedule, active, lastSync
- **Queries:** `packages/db/src/queries/external-sources.ts` -- CRUD completo con cifrado AES-256-GCM de credenciales (5 funciones: create, list, listActiveToSync, updateLastSync, delete)
- **Tests:** `packages/db/src/__tests__/external-sources.test.ts` -- 12 tests cubriendo crypto, CRUD, scheduling, y aislamiento por usuario
- **Archive:** `_archive/actions/external-sources.ts` -- server actions viejas (pre next-safe-action, recuperables)
- **Archive:** `_archive/workers/external-sync.ts` -- worker skeleton con master lock Redis (recuperable como referencia)
- **BullMQ:** `apps/web/src/lib/queue.ts` -- ya tiene ingestion queue con retry + backoff exponencial
- **Worker pattern:** `apps/web/src/workers/messaging-notifications.ts` -- patron BullMQ ya establecido en el proyecto

### Decisiones de diseno

1. **Un conector = un modulo en `apps/web/src/lib/connectors/`** -- no un package separado. Los conectores son logica de la web app, no reutilizable fuera de ella.
2. **BullMQ para scheduling** -- no setInterval como el worker archivado. Ya tenemos la infra.
3. **Cifrado ya resuelto** -- `encryptCredentials`/`decryptCredentials` en external-sources.ts ya usan AES-256-GCM con SYSTEM_API_KEY.
4. **OAuth callbacks via Next.js API routes** -- `/api/connectors/callback/[provider]`
5. **Content hashing para change detection** -- SHA-256 del contenido del documento, guardado en una nueva tabla `sync_documents`
6. **Provider enum extendido** -- agregar `web_crawler` al schema existente

   ```typescript
   export interface DocumentMeta {
     externalId: string
     title: string
     mimeType: string
     sizeBytes?: number
     lastModified?: number        // epoch ms
     parentPath?: string           // "/folder/subfolder"
   }

   export interface DocumentContent {
     externalId: string
     title: string
     content: Buffer | string      // raw file bytes or extracted text
     mimeType: string
     sizeBytes: number
   }

   export interface ChangedDocument {
     externalId: string
     changeType: "created" | "modified" | "deleted"
     title: string
     lastModified?: number
   }

   export interface ListOptions {
     pageToken?: string
     maxResults?: number
     folderPath?: string
   }

   export interface ListResult {
     documents: DocumentMeta[]
     nextPageToken?: string
   }

   export interface IConnector {
     readonly provider: string

     /** Test credentials / connection. Throws on failure. */
     authenticate(credentials: Record<string, unknown>): Promise<void>

     /** List documents, paginated. */
     listDocuments(
       credentials: Record<string, unknown>,
       options?: ListOptions
     ): Promise<ListResult>

     /** Fetch a single document's content. */
     fetchDocument(
       credentials: Record<string, unknown>,
       externalId: string
     ): Promise<DocumentContent>

     /** Detect changes since a given timestamp. Not all connectors support this. */
     detectChanges?(
       credentials: Record<string, unknown>,
       since: number   // epoch ms
     ): Promise<ChangedDocument[]>
   }
   ```

1. Crear `registry.ts` -- registro de conectores:
   ```typescript
   import type { IConnector } from "./types"
   import type { ConnectorProvider } from "@rag-saldivia/shared"

   const connectors = new Map<string, IConnector>()

   export function registerConnector(connector: IConnector): void {
     connectors.set(connector.provider, connector)
   }

   export function getConnector(provider: ConnectorProvider): IConnector {
     const c = connectors.get(provider)
     if (!c) throw new Error(`Connector not registered: ${provider}`)
     return c
   }

   export function listRegisteredConnectors(): string[] {
     return Array.from(connectors.keys())
   }
   ```

2. Agregar `externalSyncQueue` en `queue.ts`:
   ```typescript

     connection: getBullMQConnection(),
     defaultJobOptions: {
       attempts: 3,
       backoff: { type: "exponential", delay: 30_000 },
       removeOnComplete: 50,
       removeOnFail: 200,
     },
   })

   export type ExternalSyncJobData = {
     sourceId: string

     credentials: string    // encrypted JSON (decrypted by worker)

     fullSync?: boolean     // force full re-sync
   }
   ```

   - Actualiza `lastSync` de la fuente

## Dependencias externas nuevas

**NINGUNA.** Todos los conectores usan `fetch` nativo de Bun. No se agrega
googleapis, @microsoft/microsoft-graph-client, ni ningun otro SDK.
Los OAuth flows se implementan con HTTP directo + jose (ya instalado) para HMAC.

## Variables de entorno nuevas

| Variable | Requerida | Descripcion |
|----------|-----------|-------------|
| `GOOGLE_CLIENT_ID` | No (solo si se usa Google Drive) | OAuth 2.0 client ID de Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | No (solo si se usa Google Drive) | OAuth 2.0 client secret |
| `AZURE_CLIENT_ID` | No (solo si se usa SharePoint) | Azure AD app registration client ID |
| `AZURE_CLIENT_SECRET` | No (solo si se usa SharePoint) | Azure AD app registration secret |
| `AZURE_TENANT_ID` | No (solo si se usa SharePoint) | Azure AD tenant ID (puede ser per-source) |

Las credenciales de Confluence (email + API token) y Web Crawler (URLs) se
guardan directamente en la tabla `external_sources`, cifradas con AES-256-GCM.

## Consideraciones de seguridad

1. **Cifrado de credenciales:** ya resuelto en `external-sources.ts` con AES-256-GCM usando `SYSTEM_API_KEY`
2. **OAuth state:** firmado con HMAC usando `JWT_SECRET`, expira en 10 minutos, previene CSRF
3. **Tokens en memoria:** access tokens y refresh tokens solo se descifran durante el job, nunca se loguean
4. **Rate limiting:** cada conector implementa su propio rate limiter para respetar las APIs externas
5. **Content size limits:** max 10MB por documento (configurable) para prevenir DoS
6. **Domain validation:** Web Crawler solo sigue links de dominios explicitamente permitidos
7. **robots.txt:** Web Crawler respeta robots.txt (buena ciudadania)
8. **Admin only:** todas las actions requieren `adminAction` middleware
