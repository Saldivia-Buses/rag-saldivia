## Purpose

Canonical home for event type definitions and their generated bindings.

- `pkg/events/spec/` — CUE source specs, one file per family (`notify.cue`,
  `ingest.cue`, `platform.cue`). Human-edited.
- `pkg/events/gen/<family>/` — auto-generated Go package per family with
  typed `Envelope[T]`, subject templates, schema version constants, and
  enum types. **DO NOT EDIT — regenerated by `make events-gen`.**

Introduced by Plan 26 Fase 1. Codegen tool lives at `tools/eventsgen`. See
`docs/conventions/cue.md` for the CUE dialect we use.

## How it works

1. Author edits `pkg/events/spec/<family>.cue` following the syntax in
   `docs/conventions/cue.md`.
2. Run `make events-gen` locally. This produces:
   - `pkg/events/gen/<family>/<family>.go`
   - `apps/web/src/lib/events/gen/<family>.ts`
   - `docs/events/<family>.md`
3. Commit spec + generated artifacts in the same PR.
4. CI runs `make events-validate` before build; any drift between spec and
   generated output fails the build with a clear message.

## Public Go API (per family)

Each `pkg/events/gen/<family>` package exposes, per event Type:

| Symbol | Purpose |
|--------|---------|
| `<EventName>Type` | Canonical type string (e.g., `"chat.new_message"`) |
| `<EventName>SchemaVersion` | `uint8` current schema version |
| `<EventName>Subject` | NATS subject template with `{slug}`, `{action}`, … |
| `<EventName>Payload` | Typed struct matching the CUE `payload` block |
| `<EventName>` | Alias for `spine.Envelope[<EventName>Payload]` |
| `<EnumType>` + consts | When a payload field is a string disjunction |

## Usage (publisher)

```go
import (
    "github.com/Camionerou/rag-saldivia/pkg/spine"
    notify "github.com/Camionerou/rag-saldivia/pkg/events/gen/notify"
)

env, _ := spine.New(slug, notify.ChatNewMessageType,
                    notify.ChatNewMessageSchemaVersion,
                    notify.ChatNewMessagePayload{...})
subject, _ := spine.BuildSubject(notify.ChatNewMessageSubject,
                                 map[string]string{"slug": slug})
err := spine.Publish(ctx, pub, subject, env)
```

## Usage (consumer, Fase 2)

The consumer framework lands in Fase 2 (`spine.Consume[T]`). The typed
`<EventName>` alias is what the generic is instantiated with.

## Invariants

- Specs are the source of truth. Hand-edits to `gen/` are lost on the next
  `make events-gen`.
- `events-validate` gates CI. A spec change without regeneration is a CI
  failure, never a silent drift.
- Breaking changes bump the spec's `version` and archive the old as
  `<type>_v1.cue` per `docs/conventions/cue.md`.
- Nothing in `pkg/events/gen/` imports anything other than `pkg/spine`.
